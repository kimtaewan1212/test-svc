### 기능 상세

**입력 검증** (클라이언트):
- 이메일: RFC 5322 형식 검증
- 비밀번호: 최소 1자 이상

**API 호출**:
- `POST /api/v1/auth/login`
- 요청: `{ email, password }`
- 응답: `{ accessToken, refreshToken, user }`

**성공 시**:
1. JWT 토큰을 Zustand 메모리 스토어에 저장
2. `/todos` 경로로 자동 리다이렉트
3. 할일 목록 페이지 로드

**실패 시**:
- HTTP 401: "이메일 또는 비밀번호가 올바르지 않습니다"
- HTTP 500: "서버 오류가 발생했습니다"

**화면 전환**:
- 회원가입 링크 클릭 → WF-002
- 로그인 성공 → WF-003 (할일 목록)

### 기능 상세

**필터링**:
- 카테고리: 전체 또는 특정 카테고리 선택
- 상태: 전체 / 완료 / 미완료
- 기간: 전체 / 오늘 / 지난 기한 / 향후 기한

**API 호출** (필터 변경 시):
- `GET /api/v1/todos?category_id=X&status=Y&period=Z&page=1&limit=20`
- 쿼리 파라미터: `category_id`, `status`, `period`, `sort`, `order`, `page`, `limit`

**할일 카드 기능**:
- 클릭: WF-005 (상세/수정 화면)
- 체크박스: `PATCH /api/v1/todos/:id/done` (완료 토글)
- 수정: WF-005로 이동
- 삭제: 확인 모달 → `DELETE /api/v1/todos/:id`

**정렬**:
- 기본: due_date ASC, created_at DESC

**페이지네이션**:
- 오프셋 기반 (page, limit)
- 기본값: page=1, limit=20

**화면 전환**:
- 새 할일 버튼 → WF-004
- 할일 클릭 → WF-005
- 사이드바 카테고리 → WF-006

- 날짜: 선택됨

- `POST /api/v1/todos`

**카테고리 추가 (SC-006 참고)**:
