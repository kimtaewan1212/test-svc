# TodoListApp 도메인 정의서

## 1. 프로젝트 개요

### 목적
사용자 인증 기반의 개인화된 할일 관리 서비스를 제공하여, 사용자가 자신의 업무와 일정을 효율적으로 관리할 수 있도록 지원합니다.

### 배경
현재 시장에는 다음과 같은 문제점이 존재합니다.
- 개인별 할일을 체계적으로 관리할 수 있는 전용 앱의 부재
- 사용자 인증이 통합된 할일 관리 시스템 부족
- 카테고리별 할일 분류 및 관리 기능의 제한

TodoListApp은 이러한 문제들을 해결하기 위해 사용자 인증, 개인화된 할일 관리, 카테고리 기반 조직화를 핵심 가치로 제공합니다.

---

## 2. 도메인 모델

### 핵심 엔티티

#### 2.1 사용자 (User)
- **식별자**: UserId (고유 ID)
- **속성**:
  - Email: 사용자 이메일 (로그인 ID)
  - Password: 비밀번호
  - Name: 사용자명
  - CreatedAt: 가입 일시
  - UpdatedAt: 정보 수정 일시

#### 2.2 카테고리 (Category)
- **식별자**: CategoryId (고유 ID)
- **속성**:
  - UserId: 사용자 ID (사용자별 카테고리)
  - Name: 카테고리 이름
  - Color: 카테고리 색상 (선택사항)
  - IsDefault: 기본 카테고리 여부
  - CreatedAt: 생성 일시

#### 2.3 할일 (TodoItem)
- **식별자**: TodoId (고유 ID)
- **속성**:
  - UserId: 사용자 ID (소유권)
  - Title: 할일 제목
  - Description: 할일 설명
  - CategoryId: 할일이 속한 카테고리
  - DueDate: 종료 예정일
  - IsCompleted: 완료 여부
  - CreatedAt: 생성 일시
  - UpdatedAt: 수정 일시

## 3. 핵심 비즈니스 규칙

### 3.1 사용자 관리
- 사용자는 회원가입을 통해 시스템에 등록되어야만 서비스를 이용할 수 있다.
- 사용자 인증(로그인)이 완료된 사용자만 개인 정보 수정 및 할일 관리가 가능하다.
- 사용자는 자신의 개인 정보(이름, 비밀번호 등)를 언제든지 수정할 수 있다.

### 3.2 할일 관리
- 할일은 반드시 특정 사용자에 소속되어야 한다 (데이터 격리).
- 인증된 사용자는 자신의 할일만 등록, 조회, 수정, 삭제할 수 있다.
- 할일 등록 시 필수 항목: 제목, 카테고리, 종료 예정일
- 할일은 완료 여부(Boolean)로 진행 상태를 표현한다.

### 3.3 카테고리 관리
- 모든 사용자는 기본 카테고리(예: "일반", "업무", "개인")를 초기에 제공받는다.
- 사용자는 필요시 추가 카테고리를 자유롭게 생성할 수 있다.
- 카테고리는 사용자별로 독립적으로 관리된다 (다른 사용자와 공유 불가).
- 기본 카테고리는 삭제할 수 없다.

### 3.4 조회 및 필터링
- 할일 목록은 다음 조건으로 필터링할 수 있다:
  - **카테고리별**: 특정 카테고리에 속한 할일만 조회
  - **완료 여부별**: 완료된 할일, 미완료 할일, 전체 할일
  - **종료 예정일별**: 오늘, 지난 기간, 향후 기간 등으로 필터링

## 4. 주요 유스케이스

### 4.1 사용자 관리 유스케이스
- **회원가입**: 신규 사용자가 이메일과 비밀번호를 입력하여 계정 생성
- **로그인**: 사용자가 이메일과 비밀번호로 인증
- **개인 정보 수정**: 인증된 사용자가 자신의 정보(이름, 비밀번호) 변경

### 4.2 할일 관리 유스케이스
- **할일 등록**: 사용자가 제목, 설명, 카테고리, 종료 예정일을 입력하여 새 할일 생성
- **할일 조회**: 사용자가 자신의 할일 목록 조회 (필터링 조건 적용 가능)
- **할일 수정**: 사용자가 등록된 할일의 정보 업데이트
- **할일 삭제**: 사용자가 할일 제거
- **완료 표시**: 사용자가 할일 완료 여부 토글

### 4.3 카테고리 관리 유스케이스
- **카테고리 조회**: 사용자가 자신의 카테고리 목록 조회
- **카테고리 추가**: 사용자가 새로운 카테고리 생성
- **카테고리 수정**: 사용자가 카테고리 정보(이름, 색상) 변경
- **카테고리 삭제**: 사용자가 기본 카테고리가 아닌 카테고리 삭제

## 5. 용어 정의 (Ubiquitous Language)

| 용어 | 정의 |
|------|------|
| **사용자 (User)** | 시스템에 가입하고 할일을 관리하는 주체 |
| **인증 (Authentication)** | 사용자가 자신의 신원을 증명하는 프로세스 (로그인) |
| **할일 (TodoItem)** | 사용자가 수행해야 할 업무 또는 작업의 단위 |
| **카테고리 (Category)** | 할일을 논리적으로 분류하기 위한 그룹 |
| **기본 카테고리** | 시스템에서 모든 사용자에게 자동으로 제공하는 초기 카테고리 |
| **종료 예정일 (DueDate)** | 할일이 완료되어야 할 목표 날짜 |
| **완료 여부 (IsCompleted)** | 할일의 진행 상태를 나타내는 Boolean 값 (완료/미완료) |
| **데이터 격리 (Data Isolation)** | 각 사용자의 데이터가 다른 사용자와 완전히 분리된 상태 |
| **필터링 (Filtering)** | 특정 조건에 따라 할일 목록을 선택적으로 조회하는 기능 |
